# 蓝牙真机联调清单

## 测试前

1. 手机系统蓝牙已打开。
2. 播放器设备已开机，且没有被其他手机连接。
3. 微信开发者工具使用真机预览，不用模拟器验证蓝牙。
4. 首页进入“调试”页，先点击“清空日志”。

## 连接测试

1. 从引导页点击“开始扫描”。
2. 只选择真实播放器设备，不选择手表、耳机、电脑等其他 BLE 设备。
3. 连接成功后进入首页。
4. 首页“调试”页检查：
   - 服务 UUID 中应包含 `FFE5`
   - 特征值 UUID 中应包含 `FFE9` 和 `FFE8`
   - 日志中应看到 `TX` 或 `RX`

## 控制测试

1. 点击播放/暂停，检查是否有 `TX` 日志。
2. 点击上一首/下一首，检查是否发送 `0x04` 音乐控制。
3. 选择歌曲，检查是否先发送场景更新，再发送音乐开关。
4. 修改声道并保存，检查是否发送 `0x6C` 场景修改。
5. 新增/删除情景，检查是否发送 `0x6C` 增加/删除。

## 出错时看哪里

1. 提示“未找到 FFE5”：大概率连接错设备，或播放器没有广播目标服务。
2. 提示“未找到 FFE9/FFE8”：设备服务找到了，但协议特征值不匹配。
3. 提示“蓝牙连接已断开”：设备距离过远、关机、或被系统断开。
4. 提示“写入失败/超时”：先靠近设备，再清空日志重试一次。
5. 把调试页最近日志截图发给开发者分析。
